## 1️⃣ Prompt 进化史：一个任务，四种写法

```python
# 假设你用 OpenAI / Claude API（用任何 LLM 都行）
# pip install openai

task = "将以下英文翻译成中文：'The cat sat on the mat.'"

# Zero-shot: 直接问
prompt_zero = task

# Few-shot: 给几个例子
prompt_few = """将英文翻译成中文：
English: The dog is running.
中文: 狗在跑。

English: I love programming.
中文: 我喜欢编程。

English: The cat sat on the mat.
中文:"""

# Chain-of-Thought (CoT): 让模型展示推理过程
prompt_cot = """逐步翻译以下英文：
步骤1: 识别主语 → 'The cat' → '猫'
步骤2: 识别动词 → 'sat' → '坐'
步骤3: 识别介词短语 → 'on the mat' → '在垫子上'
步骤4: 组合 → '猫坐在垫子上'

现在翻译：'The bird flew over the house.'
步骤1:"""

# 结构化 Prompt（推荐！）
prompt_structured = """# 角色
你是一个专业的英中翻译专家。

# 任务
将给定的英文翻译成流畅、自然的中文。

# 要求
- 保持原文语义准确
- 使用地道的中文表达
- 只输出翻译结果，不要解释

# 输入
The cat sat on the mat.

# 输出"""
```

## 2️⃣ 结构化 Prompt 模板

```markdown
# 角色
你是一个 [专家身份]，专精于 [领域]。

# 背景
[为什么需要这个任务]

# 任务
[具体要做什么]

# 要求
- [约束条件 1]
- [约束条件 2]

# 输出格式
[期望的输出格式，用 JSON/Markdown/代码块]

# 示例（可选）
输入: [示例输入]
输出: [示例输出]

# 当前输入
[实际要处理的内容]
```

## 3️⃣ Chain-of-Thought (CoT) 实战

```python
# 数学推理：不用 CoT vs 用 CoT

question = "小明有 15 个苹果，给了小红 3 个后，又买了 8 个。现在他有几个？"

prompt_no_cot = f"回答问题：{question}"

prompt_with_cot = f"""逐步推理并回答问题。

示例：小华有 10 个橘子，吃了 2 个后买了 5 个。现在有几个？
推理：10 - 2 = 8，8 + 5 = 13。答案：13 个。

现在回答：{question}
推理："""

# 省流：CoT 把准确率从 ~40% 提升到 ~85%（在复杂推理任务上）
print("💡 CoT 的核心：让模型'说出思考过程'，而不是直接猜答案")
```

## 4️⃣ 高级技巧速览

```python
# === Self-Consistency ===
# 同一个问题跑 5 次（temperature > 0），取多数答案
# 对数学推理和选择题特别有效

# === Tree-of-Thought (ToT) ===
# 模型生成多个"思路分支" → 评估每个分支 → 深入最有希望的分支
# 对需要规划的任务（写代码、做方案）有效

# === ReAct ===
prompt_react = """你可以使用以下工具：
- search(query): 搜索互联网
- calculate(expr): 计算数学表达式

用以下格式回答：
Thought: [你的推理]
Action: [工具名](参数)
Observation: [工具返回结果]
... (重复直到有答案)
Final Answer: [最终答案]

问题：2024 年诺贝尔物理学奖得主的年龄是多少？"""

# === 少样本动态选择 ===
# 从示例库中检索最相似的 k 个示例，动态拼入 Prompt
# 比固定示例效果好得多
```

    scores = []
