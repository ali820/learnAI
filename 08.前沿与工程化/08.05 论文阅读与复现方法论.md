# 论文阅读与复现方法论

> 从"看了等于白看"到"看完能写进代码里"。论文阅读是区分工程师和研究者的分水岭。

---

## 🎯 练习目标

1. 掌握三遍阅读法
2. 学会找论文、筛论文、精读论文
3. 建立自己的论文笔记系统
4. 完成一次从"读"到"复现"的完整闭环

---

## 1️⃣ 三遍阅读法（Keshav 法）

```
第一遍（5-10 分钟）：快速扫描
  ✓ 标题 + 摘要 + 结论
  ✓ 看图表（不读正文）
  ✓ 决定：这篇文章值不值得深读？

第二遍（1 小时）：理解贡献
  ✓ 读引言（背景 + 贡献）
  ✓ 读方法（核心算法/架构）
  ✓ 读实验（主要结果表格）
  ✓ 可以跳过证明细节
  ✓ 结束后能向别人讲清楚：做了什么，为什么有效

第三遍（2-4 小时）：深度重现
  ✓ 读每个公式、每个实现细节
  ✓ 在脑海中"虚拟复现"整个实验
  ✓ 发现论文的不足和改进空间
```

## 2️⃣ 找论文：信息源金字塔

```
层 1 — 日常追踪
  - HuggingFace Daily Papers: https://huggingface.co/papers
  - Twitter/X: 关注 @_akhaliq @arankomatsuzaki
  - Arxiv Sanity: https://arxiv-sanity-lite.com/

层 2 — 领域综述
  - Papers With Code: 看 SOTA 排行榜 + 对应论文
  - Semantic Scholar: 看引用网络
  - Connected Papers: 看论文之间的引用关系图

层 3 — 深度挖掘
  - 从一篇经典论文的 References 顺藤摸瓜
  - 在 Papers With Code 上找"improves upon"链
  - GitHub Awesome Lists
```

## 3️⃣ 论文笔记模板实战

```markdown
# [论文标题]
## 元信息
- 作者/机构/年份/会议
- 链接/code

## 一句话总结
> 用一句话说清楚这篇论文的核心贡献

## 问题
- 之前的 SOTA 有什么不足？
- 本文要解决什么问题？

## 方法
- 核心架构/算法（画图！）
- 关键公式（只记最重要的 1-2 个）

## 实验结果
- 主要指标：在 XX 数据集上达到 YY
- 和 baseline 的对比：提升了多少？
- Ablation Study：哪个组件最重要？

## 我的思考
- 最大的亮点是什么？
- 有什么局限性？
- 如果我来做，会在哪里改进？
```

## 4️⃣ 复现的五个层级

```
Level 1: 跑通官方代码 → 复现表格数据
Level 2: 看懂核心算法 → 不看代码手写伪代码  
Level 3: 不看官方实现 → 从零写出能跑通的版本
Level 4: 在自己的数据上复现 → 解决适配问题
Level 5: 改进 → 改一个组件并证明效果更好
```

```python
# 复现检查清单
checklist = """
□ 下载了论文 + 附录
□ 找到了官方代码或开源实现
□ 理解了每个公式的变量含义
□ 画出了算法流程图
□ 在自己的环境里跑通了训练
□ 复现结果与论文报告误差 < 5%
□ 记录了自己遇到的坑和解决方法
"""
```
