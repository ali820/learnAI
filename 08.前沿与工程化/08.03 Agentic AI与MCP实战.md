## 1️⃣ Agent 的进化路线

```
Level 0: Chatbot — 一问一答，没有工具
Level 1: Tool Use — 能调用预定义工具（搜索/计算）
Level 2: Planning — 能自主分解任务、规划步骤
Level 3: Multi-Agent — 多个 Agent 协作
Level 4: Autonomous — 长期自主运行，自我改进
```

```python
# Level 0 → 1 的关键：把工具描述注入 System Prompt
system_prompt = """你是一个 AI 助手，可以使用以下工具：

1. search(query: str) — 搜索互联网
2. calculator(expr: str) — 计算数学表达式
3. send_email(to: str, subject: str, body: str) — 发送邮件

使用格式：
Thought: 我需要做什么
Action: 工具名(参数)
"""

# Level 1 → 2 的关键：让 Agent 先写计划，再执行
planning_prompt = """遇到复杂任务时，先输出一个计划：
Plan:
1. 搜索相关信息
2. 分析搜索结果
3. 如果需要计算，使用计算器
4. 综合信息给出答案

然后逐步执行计划..."""
```

## 2️⃣ 🔥 MCP：Agent 的"USB 协议"

```
传统方式：                         MCP 方式：
每个工具写一个 wrapper             统一协议，即插即用

Agent → 自定义代码 → API1           Agent → MCP Client → Server1 → API1
Agent → 自定义代码 → API2                          → Server2 → API2
Agent → 自定义代码 → API3                          → Server3 → API3

MCP = 让任何 LLM 应用都能用统一方式接入任何数据源/工具
```

```python
# MCP 的核心概念（伪代码）
"""
MCP Server 暴露的能力：
  - Resources: "我可以提供什么数据"（文件、数据库、API）
  - Tools: "我可以执行什么操作"（搜索、计算、发邮件）
  - Prompts: "我推荐用什么提示模板"

MCP Client (Agent 端):
  1. 发现 Server 的工具列表
  2. 把工具描述注入 System Prompt
  3. LLM 选择工具 → Client 调用 Server → 返回结果 → LLM 继续
"""

# 用 FastMCP 快速搭建一个 Server
# pip install mcp
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("My Tools")

@mcp.tool()
def get_weather(city: str) -> str:
    """获取指定城市的天气信息"""
    # 实际项目中接天气 API
    return f"{city}: 晴, 22°C"

@mcp.tool()
def search_knowledge_base(query: str) -> list[str]:
    """在知识库中搜索"""
    return [f"关于'{query}'的文档1", f"关于'{query}'的文档2"]

# 启动 Server: mcp run
```

    "never_execute": ["rm -rf", "DROP TABLE", "os.system"],
