# 🧠 学习思维建设

> AI 学习是一场马拉松，不是短跑。以下四件事决定了你能跑多远。

---

## 1️⃣ 学习如何学习

### 核心原则

**专注模式 vs 发散模式**
- 🔴 专注模式：25-50 分钟高强度学习（番茄钟）
- 🟢 发散模式：散步、洗澡、睡觉时大脑自动整理信息
- 每天必须有两者交替

> 💡 遇到卡住的难题？起身走 5 分钟，让大脑切换到发散模式。

### 间隔重复
- 学完一个概念后，1 天、3 天、1 周、1 月后分别回顾
- Obsidian 插件 [[01.编程基础/01.90 Obsidian插件配置指南|Spaced Repetition]] 可辅助

### 主动回忆
- 合上笔记本，试着复述刚才学到的内容
- 比反复阅读有效 3 倍以上

## 2️⃣ 提问的智慧

### 好问题 vs 坏问题

| ❌ 坏问题 | ✅ 好问题 |
|----------|---------|
| "梯度下降为什么不行？" | "我的梯度下降在第 50 轮后 loss 震荡，lr=0.1，数据已标准化。这是什么原因？" |
| "PyTorch 好难，怎么学？" | "我刚学 PyTorch，在 DataLoader 的 collate_fn 上卡住了，我用的是变长序列数据。" |

### 提问前自检（STFW 原则）
1. 先搜索（Google / StackOverflow / 项目 Issue）
2. 提供最小复现代码
3. 描述期望行为 vs 实际行为
4. 列出已尝试的解决方法

## 3️⃣ 卡片笔记法实践

> 核心理念：**不是按主题归档，而是让知识在链接中生长。**

### 三张卡片

| 卡片类型 | 用途 | 示例 |
|---------|------|------|
| **闪念卡片** | 快速记录想法 | "梯度下降和牛顿法有什么区别？" |
| **文献卡片** | 记录书中/论文中的观点（用自己的话） | "西瓜书说：过拟合 = 模型把噪声也学到了" |
| **永久卡片** | 一个独立的想法，能和其他卡片链接 | "梯度下降之所以有效，是因为负梯度方向是局部最优下降方向" |

### 在 Obsidian 中实践

```markdown
# 每篇笔记遵循 Feynman 原则：
1. 用自己的话解释（假装讲给 5 岁小孩）
2. 找出知识缺口，回去查资料补上
3. 用类比和故事包装
4. 链接到至少 3 篇其他笔记
```

### 每日链接检查
打开 Obsidian 的关系图谱，检查：
- 有没有"孤儿笔记"（没有任何入链）？
- 有没有"死胡同笔记"（只有入链没有出链）？

## 4️⃣ 个人学习 OKR

> OKR = Objectives and Key Results。目标 + 可量化的关键结果。

### Phase 1 OKR 模板

```markdown
## Objective 1：能独立用 Python 处理真实数据
- KR1: 完成 01.02-01.05 全部练习
- KR2: 独立完成 P1-数据分析可视化 项目
- KR3: 用 NumPy 从零实现线性回归（不查任何教程）

## Objective 2：理解 AI 所需的核心数学
- KR1: 能徒手写出梯度下降的 Python 代码
- KR2: 能手工计算一个 3 样本的贝叶斯分类
- KR3: 理解矩阵乘法的几何意义并可视化

## Objective 3：建立学习系统
- KR1: Obsidian 每日笔记连续 7 天
- KR2: 笔记库有 20+ 篇，无孤儿笔记
- KR3: Git 备份连续 7 天
```

### 每周 Review 问题

1. 本周学到了什么概念？能教会别人吗？
2. 哪个地方最卡？下周怎么突破？
3. 笔记链接网络增长了吗？
4. 下周最重要的 3 件事是什么？
