## 1️⃣ 混合精度训练

```python
import torch

# FP32: 4 bytes × 70 亿参数 = 28GB（存不下！）
# FP16: 2 bytes × 70 亿参数 = 14GB（精度损失）
# BF16: 2 bytes × 范围同 FP32（推荐）

# 一行代码开启混合精度
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()
for batch in dataloader:
    with autocast():  # 自动混合精度
        output = model(batch)
        loss = criterion(output, labels)
    
    scaler.scale(loss).backward()  # 缩放 loss 防止下溢
    scaler.step(optimizer)
    scaler.update()

print("💡 混合精度 = 前向用 FP16 加速 + 关键操作保留 FP32 + loss 缩放")
```

## 2️⃣ Flash Attention：把 O(n²) 的内存变成 O(n)

```python
# 标准 Attention:
#   S = Q @ K^T          → 显存 O(n²)
#   A = softmax(S)       → 显存 O(n²)  
#   O = A @ V            → 显存 O(n²)
# 问题：注意力矩阵 S 需要完整存到显存

# Flash Attention:
#   把 Q,K,V 分块，逐块计算 softmax
#   → 不需要存完整的注意力矩阵
#   → 显存从 O(n²) 降到 O(n)
#   → 速度提升 2-4x

# pip install flash-attn
# from flash_attn import flash_attn_func
# output = flash_attn_func(Q, K, V, causal=True)

print("💡 Flash Attention 是 2023 年最重要的推理突破之一")
print("   GPT-4 / Claude 能用 128K+ 上下文，Flash Attention 功不可没")
```

## 3️⃣ KV Cache：推理加速的关键

```python
# 自回归生成时：
# 第 1 步：算 Q₁, K₁, V₁ → 输出 token₁
# 第 2 步：算 Q₂, K₂, V₂ → 但 K₁, V₁ 已经在第 1 步算过了！

# Naive: 每步重新算所有 K, V → O(n²)
# KV Cache: K, V 算一次，缓存起来 → O(n)

# vLLM 的 PagedAttention：把 KV Cache 分页管理（像操作系统的虚拟内存）
# 避免显存碎片，提升吞吐 10-20x

print("💡 没有 KV Cache → GPT 生成 1 个 token 慢如蜗牛")
print("   有 KV Cache → 逐 token 生成 O(n) 而非 O(n²)")
```

## 4️⃣ 模型量化：把 16-bit 压到 4-bit

```python
# 量化的本质：用更少的 bit 存权重
# FP16: 每个权重 2 bytes
# INT8: 每个权重 1 byte → 减半
# INT4: 每个权重 0.5 byte → 四分之一

# 71B 模型显存对比：
print("LLaMA-70B 显存需求:")
print(f"  FP32: {70 * 4:.0f} GB")
print(f"  FP16: {70 * 2:.0f} GB")
print(f"  INT8: {70 * 1:.0f} GB")
print(f"  INT4: {70 * 0.5:.0f} GB  ← 一张 3090 就能跑！")

# GGUF (llama.cpp): CPU 推理
# GPTQ: GPU 量化，精度损失小
# AWQ: 激活感知量化，比 GPTQ 更好
# bitsandbytes: HuggingFace 集成的最简单方案

from transformers import BitsAndBytesConfig
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_compute_dtype=torch.float16,
    bnb_4bit_use_double_quant=True,  # 二次量化，再省 0.4 bit/param
)
```

## 5️⃣ 显存分析实战

```python
# 模型显存 = 参数 + 梯度 + 优化器状态 + 激活值

def estimate_memory(params_billion, batch_size, seq_len, hidden_dim):
    # 参数（FP16）
    params_mem = params_billion * 2  # GB
    
    # 梯度（FP16）
    grad_mem = params_billion * 2
    
    # Adam 优化器状态（FP32 × 2）
    optim_mem = params_billion * 4 * 2  # m + v
    
    # 激活值（估算）
    activations_mem = batch_size * seq_len * hidden_dim * 2 / 1e9
    
    total = params_mem + grad_mem + optim_mem + activations_mem
    return total

mem = estimate_memory(7, batch_size=32, seq_len=2048, hidden_dim=4096)
print(f"7B 模型训练显存估算: {mem:.1f} GB")
print(f"7B 模型推理显存估算: {7*2:.0f} GB (只需参数)")

# 显存优化技巧
print("\n💡 显存不够时：")
print("   1. Gradient Checkpointing → 牺牲 20% 速度换 50% 显存")
print("   2. Gradient Accumulation → 模拟大 batch")
print("   3. CPU Offloading → 把优化器状态放 CPU")
print("   4. DeepSpeed ZeRO → 分布式切分优化器/梯度/参数")
```
