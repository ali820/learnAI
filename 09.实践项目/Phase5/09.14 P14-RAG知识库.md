# P14-RAG 知识库

> 搭建一个完整的 RAG 知识库问答系统。把 Obsidian 学习笔记本身作为知识源！

---

## 🎯 项目目标

1. 实现文档 → 分块 → 向量化的完整 pipeline
2. 搭建带检索和生成的 QA 系统
3. 构建 Gradio/Streamlit UI
4. 评估 RAG 的回答质量

---

## 📦 数据

你自己的 Obsidian 笔记！把所有 .md 文件作为知识库。

---

## 📋 任务步骤

### Step 1：文档处理
- [ ] 读取 vault 中所有 .md 文件
- [ ] 按段落/标题分块
- [ ] 生成 Embedding 存入 Chroma/FAISS

### Step 2：检索+生成
- [ ] 实现检索（Top-K + 相似度阈值过滤）
- [ ] 构建 Prompt 模板
- [ ] 对接 LLM API

### Step 3：UI
```python
import gradio as gr
def ask(question):
    docs = retrieve(question)
    answer = generate(question, docs)
    return answer, docs

gr.Interface(fn=ask, inputs="text", outputs=["text", "text"]).launch()
```

### Step 4：评估
- [ ] 写 20 个测试问题
- [ ] 检查回答是否基于文档（有无幻觉）
- [ ] 调整 chunk_size 和检索策略优化

---

## 🏆 验收标准

- [ ] 能用 UI 提问并基于笔记内容回答
- [ ] 回答有明确的文档来源引用
- [ ] 20 题测试中幻觉率 < 10%
